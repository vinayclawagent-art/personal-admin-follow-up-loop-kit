## Purpose

Use this card only after a real or sanitized trial has been run. It converts future evidence into an explicit promote / pilot-only / iterate / hold decision before public README claims or the package-local skill draft are changed.

## Pre-decision gates

- [ ] Low-risk or sanitized thread is selected.
- [ ] `Refund Follow-Up Trial Packet.md` is completed without live private data in public files.
- [ ] `Refund Follow-Up Trial Evidence Index.md` maps every proposed claim to proof.
- [ ] Draft messages, approval checkpoints, vendor responses, and resolution status are linked or summarized safely.
- [ ] Privacy and consent risks are reviewed before any automation/promotion wording changes.

## Decision options

- **Promote** — safe and repeatable enough to install the skill and update public README language.
- **Pilot-only** — useful for personal/internal use, but keep claims narrow and skill package-local.
- **Iterate** — evidence shows promise, but approval/privacy/escalation rules need another loop.
- **Hold** — evidence is too weak, too case-specific, or privacy risk is unresolved.
